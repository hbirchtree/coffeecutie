Detailed Description

Images that may be read and written to by shaders On GL 4.2+ this will be using the actual shader image functionality.

On GL 4.1 and lower, this might be implemented with a fragment shader and a separate read/write texture, using fragment location outputs for redirection to textures. Copies back to original texture will be done by CopyTexSubImage*
